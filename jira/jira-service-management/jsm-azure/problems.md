\# Problem Management

Identify root causes of recurring incidents across Azure and AWS infrastructure and implement permanent fixes to prevent future occurrences



\## Problem Triggers

* Repeated incidents for the same VM / EC2 instance
* Frequent performance alerts (CPU, memory, disk)
* Recurring network or access failures
* Multiple incidents with the same root cause
* Repeated service outages (Nginx / web services)



\## Problem Sources

* Jira Service Management incident trends
* Azure Monitor alert patterns
* Azure Log Analytics insights
* AWS CloudWatch metrics and alarms
* AWS EC2 status check failures



\## Problem Workflow

1. Problem record Creation from related incidents
2. Root cause analysis (RCA)
3. Identify permanent fix
4. Implementation via approved change
5. Verification and monitoring
6. Problem Closure with documentation



\## Example Problems

* Repeated high CPU utilization on Linux VM (Azure/AWS)
* Nginx service stopping intermittently
* Misconfigured NSG or Security Group rules
* Insufficient VM sizing causing performance degradation
* Disk space exhaustion due to log growth



\## Root Cause Analysis Techniques

* Azure Metrics
* Azure Log Analytics Queries
* AWS CloudWatch metrics analysis
* Activity Logs and CloudTrail review
* Configuration review (NSG/Security Groups)
* OS-Level investigation via SSH / RDP



\## Tools Used

* Azure Monitor
* Azure Log Analytics Workspace
* AWS CloudWatch
* AWS CloudTrail
* Jira Service Management
* Powershell and AWS CLI



\## Output

* Document RCA
* Preventive and corrective actions
* Linked incidents and changes
* Improved system stability
