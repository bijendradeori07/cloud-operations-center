\# Jira Software – Azure Delivery Project



This directory represents a Jira Software (JSW) project used to track and manage Azure cloud delivery work in a structured, engineering-driven manner. The project mirrors real-world Agile delivery used by Cloud Operations and DevOps teams.



---



\## Purpose



The objective of this Jira Software project is to:



* Plan and track Azure infrastructure delivery
* Break down cloud work into Epics, Stories, Tasks, and Sub-tasks
* Maintain clear traceability between delivery work and actual Azure resources
* Support Agile execution with operational awareness



---



\## Scope of Work



This Jira Software project covers Azure-related delivery such as:



* Azure VM provisioning (Linux \& Windows)
* VNet and subnet design
* NSG configuration and validation
* RBAC validation
* Monitoring and Log Analytics enablement
* Backup and Recovery Services Vault setup
* Nginx deployment on Linux VMs
* Operational validation using PowerShell, SSH, and RDP



---



\## Jira Work Item Structure



The project follows standard Jira Software hierarchy:



* Epics → High-level delivery initiatives
* Stories → User or system requirements
* Tasks → Implementation-level work
* Sub-tasks → Granular execution steps



Each folder in this directory reflects this hierarchy.



---



\## Folder Structure



* `epics/`      → High-level Azure delivery initiatives
* `stories/`    → Functional or technical requirements
* `tasks/`      → Implementation activities
* `subtasks/`   → Step-by-step execution work
* `jsw-azure-architecture.md` → Delivery architecture and integration context
